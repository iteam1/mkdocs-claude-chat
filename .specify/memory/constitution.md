# mkdocs-claude-chat Constitution

> Conventions, structure, and principles for this plugin.
> Derived from best practices in `mkdocs-llmstxt` and `markdown-exec`.

---

## I. Purpose

`mkdocs-claude-chat` is a MkDocs plugin that injects a Claude-powered chatbot widget into every page of a documentation site. The chatbot uses `claude-agent-sdk-python` (which wraps the Claude Code CLI) to answer questions, using `/llms.txt` and per-page `.md` files generated by `mkdocs-llmstxt` as its knowledge source.

**Deployment assumption:** The environment always has the `claude` CLI installed and authenticated.

## IV. Plugin Design

### Config schema

Inherit from `mkdocs.config.base.Config`, use `mkdocs.config.config_options` for field types.

```python
from mkdocs.config.base import Config
import mkdocs.config.config_options as mkconf

class _PluginConfig(Config):
    enabled       = mkconf.Type(bool, default=True)
    model         = mkconf.Type(str, default="claude-opus-4-5")
    system_prompt = mkconf.Optional(mkconf.Type(str))
    llmstxt_url   = mkconf.Optional(mkconf.Type(str))  # auto-detected if not set
    chat_title    = mkconf.Type(str, default="Ask Claude")
    position      = mkconf.Type(str, default="bottom-right")
```

### Plugin class rules

```python
class MkdocsClaudeChatPlugin(BasePlugin[_PluginConfig]):
    # Declare state as class-level type hints
    _llmstxt_url: str
    _site_dir: str

    def on_config(self, config: MkDocsConfig) -> MkDocsConfig | None: ...
    def on_post_build(self, *, config: MkDocsConfig, **kwargs) -> None: ...
    def on_page_context(self, context, /, *, page: Page, **kwargs): ...
```

- Always accept `**kwargs` in hook signatures for forward compatibility.
- Populate instance state in `on_config`; never in `__init__`.
- Return modified object or `None` to signal no-op.
- Respect `self.config.enabled` вЂ” short-circuit all hooks when `False`.

### Asset injection

Inject CSS/JS in `on_config` by appending to `config["extra_css"]` / `config["extra_javascript"]` and copying bundled files to `site_dir` in `on_post_build`.

## V. Chat Backend

Use `ClaudeSDKClient` from `claude-agent-sdk-python`. Define custom `@tool` functions in `_internal/tools.py` that fetch from `/llms.txt` and per-page `.md` URLs.

```python
@tool("fetch_llmstxt", "Get the documentation index", {})
async def fetch_llmstxt(args): ...

@tool("fetch_page", "Get the content of a documentation page", {"url": str})
async def fetch_page(args): ...
```

**Rules:**
- Tools fetch from HTTP URLs вЂ” never from raw filesystem paths.
- Keep tools stateless; pass config via closure.
- The chat backend runs as a local server (FastAPI/aiohttp) spawned by the plugin or invoked via JS fetch from the browser.

## VII. Type Hints

- `from __future__ import annotations` in every module.
- Built-in generics: `dict[str, str]`, `list[str]`, `tuple[int, ...]`.
- `TYPE_CHECKING` guard for expensive or circular imports.
- All public functions fully annotated.
- Include `py.typed` marker.

## VIII. Logging

Use a logger adapter that prefixes the plugin name, writing to `mkdocs.plugins.*` namespace.

```python
def get_logger(name: str) -> _PluginLogger:
    logger = logging.getLogger(f"mkdocs.plugins.{name}")
    return _PluginLogger(name.split(".", 1)[0], logger)
```

Use `_logger = get_logger(__name__)` at module level in each `_internal` module.

## IX. Testing

- **Framework:** `pytest` + `pytest-cov` + `pytest-asyncio`
- Use real `MkDocsConfig` objects вЂ” no mocks for MkDocs internals.
- Mock `ClaudeSDKClient` for chat backend tests (avoid real CLI calls in CI).
- **Coverage target: в‰Ґ 90%** (`fail_under = 90` in `coverage.ini`)

### `conftest.py` pattern

```python
@pytest.fixture(name="mkdocs_conf")
def fixture_mkdocs_conf(tmp_path: Path) -> MkDocsConfig:
    conf = MkDocsConfig()
    conf.load_dict({
        "site_name": "Test",
        "site_url": "https://example.org/",
        "site_dir": str(tmp_path / "site"),
        "docs_dir": str(tmp_path / "docs"),
        "plugins": {"claude-chat": {}},
    })
    Path(conf.docs_dir).mkdir()
    assert conf.validate() == ([], [])
    return conf
```

## XIII. Versioning & Deprecation

- Semantic versioning: `MAJOR.MINOR.PATCH`
- Mark legacy code: `# YORE: Bump X: Remove file/block.`
- `warnings.warn(..., DeprecationWarning, stacklevel=2)` for deprecated public APIs
- Never remove a public API without a deprecation cycle (в‰Ґ 1 minor version)

## XIV. Non-Goals

- Supporting non-Claude LLM backends
- Providing a standalone chat server independent of MkDocs
- Offline / air-gapped operation
- Server-side conversation history storage
